最近发现一个很适合通勤时用的小工具：[abogen](https://github.com/denizsafak/abogen)。

如果你平时会囤很多 PDF、EPUB、Markdown 笔记，或者总觉得“这篇东西想看，但坐下来看的时间不够”，那这个工具就很对路。它做的事情很简单：**把这些文本资料直接转成可以听的音频。**

## 它适合用来干什么？

根据官方 README，`abogen` 支持把 `ePub`、`PDF`、普通文本、`Markdown`，甚至字幕文件转成高质量音频，并且还能生成同步字幕。[GitHub 仓库](https://github.com/denizsafak/abogen)

对我来说，它最有吸引力的点不在于“玩 TTS”，而在于它把很多原本只能坐着看的东西，变成了可以在通勤、走路、做家务时继续吸收的内容。

一个很直接的场景就是：

1. 白天把课程讲义、电子书、长文资料存下来。
2. 晚上或者周末批量转成音频。
3. 第二天路上直接听。

这样很多“没时间看”的东西，至少有机会先进入耳朵。

## 我觉得它好用的地方

我看了一下官方说明，它不只是“把文本念出来”这么简单，还有几个比较实用的点：

- 有桌面 GUI，也有 Web UI。
- 支持队列处理，可以一次加多个文件。
- 支持章节控制，能选章节，也能合并成单个文件。
- 输出格式不只一种，支持 `WAV`、`FLAC`、`MP3`、`OPUS`，也支持 `M4B`。
- 字幕生成的粒度也能调，比如按句子、按词，或者高亮式字幕。

官方还提到，Web UI 目前功能更丰富，桌面版则更偏稳定核心功能。这一点挺重要，因为很多这类工具最后都卡在“功能很多但入口混乱”，而 `abogen` 至少把桌面版和 Web 版的定位写得比较清楚。[README 的 Interfaces 部分](https://github.com/denizsafak/abogen#interfaces)

## 怎么开始

如果你是 Windows 用户，官方给的路线比较直接：

1. 先安装 `espeak-ng`。
2. 下载仓库后运行 `WINDOWS_INSTALL.bat`。

官方说明里写得很明确：这个脚本会自动处理依赖，甚至会连 Python 一起配好，你不一定需要自己手动折腾环境；当然，前提还是先把 `espeak-ng` 装好。[Windows 安装说明](https://github.com/denizsafak/abogen#windows)

如果你本来就习惯命令行，官方也给了 `uv` 的安装方式。装好以后，桌面版直接运行：

```bash
abogen
```

然后最基础的使用流程就是：

1. 把 `ePub`、`PDF`、`TXT`、`MD` 或字幕文件拖进去。
2. 选语速、声音、字幕样式和输出格式。
3. 点开始。

这套流程对“把资料变成路上能听的东西”来说已经够用了。[How to use](https://github.com/denizsafak/abogen#how-to-use)

## 一个我会怎么用的例子

如果我以后遇到一份不错的 PDF 资料，第一反应可能不是“等我哪天有空坐下来看”，而是：

- 先留 PDF 原文做精读；
- 再用 `abogen` 转一份音频版本；
- 通勤路上先听一遍，建立大概轮廓；
- 真要做笔记或者深入理解时，再回到原文。

这样资料的利用率会高很多。

对想把碎片时间尽量榨干一点的人来说，这确实是个很顺手的工具。
